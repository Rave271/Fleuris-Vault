Security controls

What was hardened

Migrated from a deliberately vulnerable banking app to a production-style architecture with real access controls, audit trails, and session management.

A07 · Authn

Authentication Hardening

Bcrypt password hashing, parameterised queries, secure cookie flags, account lockout after 5 failed attempts, and optional TOTP MFA.

A01 · Access Control

Role-Based Access

Customers can only access their own data. Admin routes check server-side role. Transfers are always scoped to the authenticated user.

A03 · CSRF

Transaction Safety

All state-changing forms use CSRF tokens. Transfers validate amount, recipient, and include a confirmation review step.

A05 · Headers

Security Headers

Content-Security-Policy, X-Frame-Options DENY, X-Content-Type-Options, and Referrer-Policy on every response.

A09 · Logging

Audit Monitoring

Structured JSON logs for every login, transfer, and access denial. Admin security center with event timeline. ELK-stack ready.

Analytics

PySpark Analytics

Batch analytics jobs read JSON logs and write endpoint popularity, login summaries, response times, and user activity to PostgreSQL.
