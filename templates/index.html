Project upgrade

Why this reference project was chosen

This site is built from the original unsafe banking reference project because it already had realistic fintech workflows: login, transfers, account statements, user records, balances, and database-backed transactions. That made it a better OWASP hardening candidate than a smaller password-only or login-only demo.

Authentication Hardening

Passwords are now hashed, login queries use parameters, sessions use safer cookie settings, and repeated failed logins trigger temporary account lockout.

Access Control

Customers can only view their own statements, admin pages are restricted, and transfers always come from the signed-in user's account.

Transaction Safety

Transfers now validate recipients and amounts, block self-transfers, include CSRF tokens, and record audit events for review.

Security Headers

The app sends practical browser protections including Content Security Policy, X-Frame-Options, X-Content-Type-Options, and Referrer-Policy.

Audit Monitoring

Login successes, failed attempts, blocked access, CSRF failures, and transfers are logged into an admin-only security center.

Actual Site Experience

The basic demo was redesigned as Fleuris Vault with a proper home page, dashboard, transfer form, statement view, customer admin area, and responsive UI.
